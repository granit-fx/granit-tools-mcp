# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/),
and this project adheres to [Semantic Versioning](https://semver.org/).

## [1.0.0] - 2026-03-23

### Added

- Local .NET 10 dotnet tool replacing the Cloudflare Worker
- SQLite FTS5 full-text search on `llms-full.txt` documentation
- Search/fetch separation: `search_docs` (lightweight) + `get_doc`
  (full content) for ~60-80% token reduction
- Automatic git branch detection from `.git/HEAD` for code index tools
- `BackgroundService` for periodic docs re-indexing (default: 4 hours)
- Graceful degradation during initial indexing
- 10 MCP tools: `search_docs`, `get_doc`, `list_patterns`,
  `search_code`, `get_public_api`, `get_project_graph`,
  `list_branches`, `list_packages`, `get_package_info`
- ADR-003: documents the migration decision

### Changed

- **BREAKING:** Transport changed from HTTP (remote Worker) to stdio
  (local dotnet tool)
- **BREAKING:** Tool `search_granit_docs` renamed to `search_docs`
- **BREAKING:** Tool `get_module_reference` replaced by `get_doc`
- Documentation source changed from `search-index.json` (manual) to
  `llms-full.txt` (auto-generated by Starlight)
- Search engine changed from custom TF-IDF (~120 lines JS) to
  SQLite FTS5 (stemming, phrase matching, prefix search)
- Code index cache changed from Cloudflare KV to in-memory with TTL

### Removed

- Cloudflare Worker deployment (`wrangler.toml`, `deploy.yml`)
- TypeScript codebase (`src/*.ts`, `package.json`, `tsconfig.json`)
- Custom TF-IDF search implementation
- `search-index.json` dependency (replaced by `llms-full.txt`)
- Rate limiting (not needed for local stdio transport)
